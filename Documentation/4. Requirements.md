| Requirement | Why We Need It |
|---|---|
| Zendesk Help Center API access (`hilfe.infleet.de/api/v2/help_center/de/`) | The knowledge base is sourced directly from Infleet's Zendesk Help Center. The API is public (no auth required for German locale). We fetch categories, sections, and paginated articles, strip HTML, chunk, embed, and store in pgvector. |
| OpenAI API Key | To call the Assistants API for chat reasoning and embeddings generation |
| Zendesk subdomain + API credentials (email + API token) for the Support API | So our Python backend can create support tickets in Infleet's Zendesk Support instance (`POST /api/v2/tickets.json`). Use a dedicated agent or API user; tickets created via the API appear in the Zendesk agent queue like any other ticket. |
| How Infleet exposes the logged-in user's data to the widget (session token, user object, or injected variables) | So we can automatically identify the user without requiring them to log in again. The injected context only needs to provide user identity (`userId`, `email`). Device serial is not required at login — the AI asks the user to identify their device during conversation before creating a support ticket. |
| Where inside the portal to inject the React chat widget | To know the exact frontend integration point inside the authenticated area |
