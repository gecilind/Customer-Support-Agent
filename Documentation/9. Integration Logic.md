# Integration Guide — Infleet AI Support Agent

The project is delivered inside your repository under `ai-support-agent/` and contains two independent parts:

- **`frontend/`** — React + Vite widget (builds to static files).
- **`backend/`** — FastAPI service (runs in Docker).

All configuration is done via `.env` files. No source code changes required.

---

## 1. Frontend

1. Open `frontend/.env.example`, copy it to `frontend/.env`, and set `VITE_API_URL` to the public URL where the backend will be reachable.
2. Run `npm install` then `npm run build` inside `frontend/`.
3. Serve the generated `dist/` folder from your static web server.
4. Reference `dist/index.html` as the entry point — it auto-loads the correct asset files. No HTML edits needed on rebuild.

> ⚠️ `VITE_API_URL` is baked in at build time. Rebuild whenever you change it.

---

## 2. Backend

1. Open `backend/.env.example`, copy it to `backend/.env`, and fill in real values for every key.
2. From inside `backend/`, run `docker build -t ai-support-backend .`.
3. Run the container with `docker run -d --name ai-support-backend -p 8000:8000 --env-file .env --restart unless-stopped ai-support-backend`.
4. Forward the following HTTP paths from your public domain to the container on port `8000` (these match the FastAPI routes in this repo):

   - `GET /health` — liveness / readiness checks
   - `POST /chat` — chat completions stream (widget)
   - `GET /conversations` — list conversations (widget)
   - `POST /conversations` — start a new conversation (widget)
   - `GET /conversations/{conversation_id}` — conversation detail
   - `GET /conversations/{conversation_id}/messages` — message history
   - `POST /create-ticket` — Jira ticket creation (standalone API; the widget escalates through `POST /chat` or voice, where ticket creation runs in-process against Jira)
   - `POST /ingest-manual` — manual / KB ingestion (admin)
   - `GET /manuals` — list ingested manual sources (admin)
   - `DELETE /manuals/{source}` — remove a manual source (admin)

   There is **no** `/search-kb` or `/check-warranty` route in this service; KB search runs inside `POST /chat`, and warranty is not exposed as a separate endpoint.

5. Forward **`WebSocket /voice-relay`** with proper upgrade headers — required for the voice session.

---

## 3. Environment Variables

All required keys are documented inline in:

- `frontend/.env.example`
- `backend/.env.example`

Copy each to `.env` in the same folder and fill in the real values.

> ℹ️ The Supabase database is already provisioned and managed on our side. The connection string for `SUPABASE_DB_URL` will be supplied separately — no Supabase setup required on your end.
